## Service
```
- cluster IP (Internal to Kubernetes Cluster)
- Node Port (It can expose port(ex:31416) of all nodes in cluster)(it can be accessed through Internet)
- LoadBalancer
```
#### service.yml
```
Service Type as Cluster-IP
By Default it will take cluster IP
```
```
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    server: nginx
spec:
  containers:
  - name: nginx
    image: nginx
    resources:
      limits:
        memory: "128Mi"
        cpu: "500m"
    ports:
      - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-svc
spec:
  selector:
    server: nginx
  ports:
  - port: 8081 # HostPort
    targetPort: 80 # Container Port
---
# This Pod is for Testing Purpose only
apiVersion: v1
kind: Pod
metadata:
  name: debug
  labels:
    server: almalinux
spec:
  containers:
  - name: almalinux
    image: almalinux
    command: ["sleep","2000"]
```

### Service type as NodePort
#### service-NP.yml
```
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    server: nginx
spec:
  containers:
    - name: nginx
      image: nginx
      ports:
      - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: NodePort
  selector:
    server: nginx
  ports:
    - protocol: TCP
      nodePort: 31317
      port: 80
      targetPort: 80
```

```
kubectl apply  -f service-NP.yml
```

### Service type as LoadBalancer
#### service-LB.yml
```
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    server: nginx
spec:
  containers:
    - name: nginx
      image: nginx
      ports:
      - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    server: nginx
  ports:
    - protocol: TCP
      port: 8081
      targetPort: 80
```
```
kubectl apply -f service-LB.yml
```
